# 😈 Critic — Беспощадный критик страховок

// turbo-all

> Жёсткий, но конструктивный аудит. Цель — найти ВСЁ плохое.
> Оценка по шкале 1-10 для каждой оси.

## Формат вызова

```text
/critic [scope]
```

**Scope:** `data` | `ui` | `ux` | `content` | `sources` | `all`

---

## Шаги

1. **Прочитай данные/UI** в указанном scope
2. **Оцени по осям** (1-10):
   - 📊 Data Accuracy — точность цен, лимитов, покрытий
   - 🔗 Source Quality — каждая цифра имеет источник? актуальный?
   - ⚖️ Fairness — нет ли перекоса в подаче (маркетинг vs анализ)?
   - 🎨 Design / UI — дизайн, читаемость таблиц
   - 👤 UX / Usability — можно ли быстро найти ответ на свой вопрос?
   - 📱 Mobile / Responsive — работает ли на телефоне?
   - 🌐 Completeness — все ли компании/пакеты/нюансы покрыты?
   - 🧪 Actionability — можно ли по сайту ПРИНЯТЬ РЕШЕНИЕ?

3. **Для каждой проблемы:**

```markdown
| Severity | Category | Issue | Location | Fix |
|----------|----------|-------|----------|-----|
```

Severity: 🔴 CRIT | 🟡 MED | 🔵 LOW

4. **Вердикт:** общая оценка + ТОП-3 критических фикса
5. **НЕ фикси сам** — только отчёт. Фиксы — через отдельную задачу.

## Специфика страхового проекта

- Каждая цена должна иметь: источник, дату проверки, пометку «ориентир» или «подтверждено»
- Отзывы должны быть привязаны к компании, а не висеть «в воздухе»
- Лайфхаки должны иметь подтверждение (чат, ссылка, опыт)
- Калькулятор должен давать осмысленные рекомендации для всех комбинаций
