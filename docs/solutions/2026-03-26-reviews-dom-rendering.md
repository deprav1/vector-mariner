# Compound Engineering Record: Отсутствие рендеринга перенесенных JS-данных в DOM

**Тема:** Панель отзывов. Баг пустой панели при разделении HTML и JS.
**Дата:** 2026-03-26

## Problem (Проблема)
После переноса массивного объекта `REVIEWS` (и добавления 576 элементов `RAW_REVIEWS`) из `index.html` во внешний файл `reviews-data.js` и создания переключателя режимов, панель отзывов открывалась абсолютно пустой. Ни табы компаний, ни список сообщений не рендерились, хотя кнопка открытия работала (добавлялся класс `.open`).

## Root Cause (Причина)
Сложилось сразу 3 фактора, которые сломали рендеринг:
1.  **Порядок загрузки скриптов:** Тег `<script src="reviews-data.js"></script>` случайно удалился при промежуточных рефакторингах HTML, поэтому объект `REVIEWS` был `undefined` в момент вызова `initReviews()`.
2.  **Синтаксическая ошибка HTML:** Кнопка пагинации `<button id="loadMoreReviews">` была ошибочно закрыта тегом `</div>` вместо `</button>`. Пакетного краша не было, но бразуер строил DOM-дерево криво, съедая часть контейнеров ниже.
3.  **Отложенная инициализация:** В старом коде стояло условие `if(panel.classList.contains('open') && !panel.dataset.init) { initReviews(); }`. Из-за ошибки `undefined` скрипт падал, `dataset.init` не ставился, но так как вкладки не рендерились, казалось, что интерфейс просто пуст.

## Solution (Решение)
1.  Восстановлен тег `<script src="reviews-data.js"></script>` в самом низу `<body>` **перед** инлайн-скриптом логики страницы, чтобы гарантировать наличие данных до инициализации.
2.  Исправлен тег: `</button>` вместо `</div>`.

## Prevention (Предотвращение на будущее)
*  При выносе данных в отдельные `*.js` файлы "data-only" формата (как `reviews-data.js`), **всегда** ставить их первыми в футере.
*  Если панель, зависящая от JavaScript-генерации (через `.innerHTML = arr.map()...`), открывается пустой, первым делом проверять консоль на предмет `ReferenceError: X is not defined` (данные не подгрузились) и инструмент Inspector на предмет разорванных/незакрытых тегов.
